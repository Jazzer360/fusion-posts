1. Program lifecycle overview

Fusion's CAM engine walks the setup tree top-to-bottom, calling these handlers in order. Each toolpath operation becomes one onSection / onSectionEnd pair, and the motion within it is delivered as onRapid / onLinear / onCircular calls. Manual NC operations interleave as onManualNC calls between sections.

2. onSection — wrap-up + new-section setup

This is the busiest handler. It detects whether a tool change / new WCS / new workplane is needed, runs the previous section's wrap-up if so, then writes the new section's preamble. The Manual NC drain runs between those two stages, which is why a Manual NC ends up between operations rather than mid-block.

Why is the wrap-up of section N done in section N+1's onSection?
  • Fusion's API gives onSectionEnd very little context — at that point the engine doesn't yet know whether the next section will need a tool change, a new WCS, or even exist. Putting the wrap-up here keeps decisions like "do we cancel the work plane?" / "do we retract Z?" / "do we shut the spindle off?" deferred until we can see both sides.
  • That's also why the Manual NC drain has to run in onSection: it must wait for the previous section's wrap-up to finish, so the Manual NC block lands cleanly between operations.

3. onSectionEnd — minimal cleanup

4. Manual NC buffering & drain customization

Fusion fires onManualNC in the natural order Manual NC operations appear in the setup tree — between onSectionEnd(prev) and onSection(next). The stock post would expandManualNC() immediately, which lands the output mid-wrap-up of the previous section. Instead we buffer (almost) everything and drain at the right moment.

Where does the drain actually run?
  • Between sections — in onSection, immediately after the previous section's wrap-up and before the new section's (operation-comment) header.
  • End of program — in onClose, after the final retract / chip-conveyor block and before M02 / RTS. Catches anything queued after the last machining section.
What does a flushed block look like in the NC file?
  • For most types:
     
    (Manual NC: Optional Stop)
    (Drill1 3) ← only if the Manual NC op has a name
    N70 M01
  • For comment-like types (no type-label header, no op-comment header — the content IS the comment):
     
    (This is the comment text)
  • Multi-line text: writeComment splits on real newlines AND the literal two-character sequence \n, so each line becomes its own (...).
